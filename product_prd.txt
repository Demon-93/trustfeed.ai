## The Problem

YouTube has an enormous amount of content. When you search for something specific — investing, Python, nutrition, law — you get dozens of videos. The problems:

- **Clickbait is everywhere.** Thumbnails and titles lie. You waste 10 minutes to realize the video has zero depth.
- **Creator credibility is invisible.** You have no idea if the person teaching you finance is a Goldman Sachs analyst or a 19-year-old with no experience.
- **No topic-specific trust.** A creator trusted in fitness may be completely unqualified in nutrition science.
- **No quality signal.** YouTube's algorithm optimizes for watch time and engagement — not accuracy, depth, or trustworthiness.

**Result:** Hours wasted on low-quality, misleading, or irrelevant content.

---

## The Solution

An AI agent where you describe what you want to learn, and it:

1. Searches YouTube for relevant videos
2. Verifies each creator's real-world credentials across social media and the web
3. Analyzes the video transcript to confirm it actually covers what you asked
4. Scores everything and returns the top ranked, trustworthy results
5. Explains exactly *why* each video was recommended

**One sentence:** Find me the most trustworthy video on [topic] — from someone actually qualified to talk about it.

## Key Differentiators

| What Exists | What We Add |
|---|---|
| YouTube search (engagement-optimized) | Trust-optimized search |
| Subscriber count as proxy for trust | Real credential verification |
| You guess if video is good | Transcript analysis confirms content quality |
| Generic results | Topic-domain matched credibility |
| No explanation | "Why we trust this" per video |

**The moat:** Nobody combines creator credential verification + transcript content analysis + topic-domain matching together. This is the gap.

- Natural language search ("best videos on compound interest for beginners")

### Smart Features
- Clickbait detection (thumbnail vs actual content mismatch)
- Red flag warnings ("Creator's expertise doesn't match this topic")

## System Architecture

```
User types query
        ↓
Next.js Frontend (Vercel)
        ↓
Next.js API Route: POST /api/search
        ↓
[Check Upstash Redis cache] → Return cached if exists
        ↓ (cache miss)
┌─────────────────────────────────────────────┐
│              AGENT PIPELINE                  │
│                                             │
│  1. Query Agent (Groq)                      │
│     → Expand query, detect domain           │
│                                             │
│  2. Search Agent (YouTube API)              │
│     → Fetch top 30 videos in parallel       │
│                                             │
│  3. For each video (parallel):              │
│     a. Creator Agent → verify credentials   │
│     b. Content Agent → analyze transcript   │
│                                             │
│  4. Ranking Agent (Groq)                    │
│     → Score, sort, generate explanations    │
└─────────────────────────────────────────────┘
        ↓
Firebase Firestore (store results)
        ↓
Upstash Redis (cache results, 1 day TTL)
        ↓
Return ranked, explained results to user
```

### Agent 5: Ranking & Explanation Agent
**File:** `lib/agents/ranking-agent.ts`

**Purpose:** Combine all scores, sort results, generate human-readable explanations.

**Trust Score Calculation:**
```typescript
function calculateTrustScore(creator, content, ytSignals): number {
  const credentialComponent = creator.credentialScore * 0.35
  const contentComponent = content.contentScore * 0.35
  const engagementComponent = calculateEngagement(ytSignals) * 0.20
  const consistencyComponent = calculateConsistency(ytSignals) * 0.10

  return Math.round(
    credentialComponent + contentComponent +
    engagementComponent + consistencyComponent
  )
}

function calculateEngagement(video): number {
  if (!video.viewCount || !video.likeCount) return 50
  const ratio = (video.likeCount / video.viewCount) * 100
  // Normalize: 4%+ like ratio = 100, 0.5% = 20
  return Math.min(Math.round(ratio * 25), 100)
}

function calculateConsistency(channel): number {
  // Based on: channel age, upload frequency, subscriber growth trajectory
  // Normalize to 0-100
  const ageScore = Math.min(channel.channelAgeYears * 10, 50)
  const uploadScore = channel.videoCount > 50 ? 50 : channel.videoCount
  return ageScore + uploadScore
}
```

**Explanation Generator (Groq):**
```
System: "Write a 2-sentence explanation of why this video is recommended.
         Be specific. Mention the creator's credentials and what makes the content good.
         Never be generic. Output plain text only."

User: "Creator: {credentialSummary}
       Content: {contentSummary}
       Score breakdown: Credentials {credentialScore}/100, Content {contentScore}/100
       Red flags (if any): {redFlags}"

Output: "Recommended because John has 12 years in institutional finance as a CFA ✓ 
         and the video covers exactly what you asked with real fund comparisons and data."
```

## Trust Score Formula

```
Trust Score (0–100) =
  (Creator Credential Score  × 0.35) +
  (Content Quality Score     × 0.35) +
  (YouTube Engagement Score  × 0.20) +
  (Channel Consistency Score × 0.10)
```

### Score Color Coding in UI
- 🟢 **70–100** — Highly Trustworthy
- 🟡 **40–69** — Moderately Trustworthy
- 🔴 **0–39** — Low Trust / Proceed with Caution

### Red Flags (shown in UI)
- `no_transcript` — Content couldn't be verified
- `expertise_mismatch` — Creator's credentials don't match this topic
- `clickbait_detected` — Title/thumbnail doesn't match content
- `no_verifiable_credentials` — No professional background found
- `low_information_density` — Video is surface-level / filler content

## Free Tier Limits & Strategy

| Service | Daily Free Limit | Risk Level |
|---|---|---|
| YouTube Data API v3 | 10,000 units/day | Medium |
| Groq API | ~14,400 req/day | Low |
| Firebase Firestore reads | 50,000/day | Low |
| Firebase Firestore writes | 20,000/day | Low |
| Upstash Redis | 10,000 cmd/day | Low |

### Caching Strategy (Stays Within Limits)

**The key insight:** Cache aggressively. Once a creator or video is analyzed, the result is stored forever (with refresh schedules). Day 2 onwards, most requests are cache hits.

```
Creator trust score  → Firestore cache, refresh every 7 days
Video analysis       → Firestore cache, refresh every 30 days
Search results       → Upstash Redis cache, 1 day TTL
Same query           → Return instantly from Redis, zero API calls
```

**YouTube API unit budget (per search):**
- Search (3 queries × 100 units) = 300 units
- Video details (30 videos × 1 unit) = 30 units
- Total per fresh search ≈ 330 units
- Allows ~30 unique fresh searches/day
- After caching kicks in: most searches cost 0 units
